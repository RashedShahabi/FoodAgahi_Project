# تأیید Schema بر اساس قوانین ساده‌سازی

## ✅ بررسی انطباق با ۴ قانون اصلی

### قانون ۱: کیف پول ساده
**وضعیت: ✅ تأیید شده**

در مدل `User`:
```prisma
walletBalance     Float     @default(0)
```

- فقط یک فیلد ساده `walletBalance` وجود دارد
- بدون جداول پیچیده لاگ مالی
- تراکنش‌ها در جدول `Transaction` به صورت ساده ذخیره می‌شوند

---

### قانون ۲: حل اختلاف ساده (بدون چت‌روم)
**وضعیت: ✅ تأیید شده**

مدل `Dispute`:
```prisma
model Dispute {
  id                String    @id @default(cuid())
  orderId           String    @unique
  order             Order     @relation(fields: [orderId], references: [id])
  
  buyerReason       String    // دلیل خریدار
  sellerResponse    String?   // پاسخ فروشنده
  
  adminDecision     String?   // تصمیم نهایی مدیر
  resolvedInFavorOf String?   // BUYER یا SELLER
  
  isResolved        Boolean   @default(false)
  
  createdAt         DateTime  @default(now())
  resolvedAt        DateTime?
}
```

**ویژگی‌ها:**
- ✅ فقط دلیل خریدار و پاسخ فروشنده
- ✅ تصمیم ساده مدیر (BUYER یا SELLER)
- ✅ بدون چت‌روم یا سیستم پیام‌رسانی پیچیده
- ✅ آپلود اسکرین‌شات در مدل `Order` (فیلد `transferScreenshot`)

## 📊 خلاصه مدل‌های دیتابیس

### 1. User (کاربران)
- احراز هویت با شماره موبایل و رمز عبور
- نقش: دانشجو یا ادمین
- چند شماره دانشجویی
- کیف پول ساده (`walletBalance`)
- شماره شبا برای برداشت

### 2. FoodAd (آگهی‌های غذا)
- اطلاعات غذا (نام، قیمت، توضیحات، عکس)
- نوع وعده (ناهار/شام)
- تاریخ انقضا خودکار
- وضعیت موجود بودن

### 3. Order (سفارش‌ها)
- خریدار و فروشنده
- محاسبات مالی دقیق (قیمت پایه، کارمزدها، سود سامانه)
- شماره دانشجویی خریدار
- اسکرین‌شات تایید انتقال
- وضعیت‌های مختلف (در انتظار، تأیید شده، تکمیل شده، لغو شده، اختلاف)

### 4. Transaction (تراکنش‌ها)
- انواع: شارژ کیف پول، خرید، فروش، برداشت، بازگشت وجه
- رفرنس درگاه پرداخت
- وضعیت تراکنش

### 5. Dispute (اختلافات)
- ساده و بدون چت‌روم
- دلیل خریدار + پاسخ فروشنده
- تصمیم مدیر

### 6. Warning (اخطارها)
- ثبت تخلفات کاربران
- دلیل و توضیحات

## 🎯 فرمول‌های مالی پیاده‌سازی شده

```typescript
// ثابت‌ها
const Cb = 0.10; // کارمزد خریدار (10%)
const Cs = 0.10; // کارمزد فروشنده (10%)

// محاسبات
const totalPaid = P + (P × Cb);           // پرداخت خریدار
const sellerReceives = P - (P × Cs);      // دریافتی فروشنده
const platformProfit = (P × Cb) + (P × Cs); // سود سامانه
```

این فرمول‌ها در فیلدهای زیر در مدل `Order` ذخیره می‌شوند:
- `basePrice` (P)
- `buyerFee` (P × Cb)
- `sellerFee` (P × Cs)
- `totalPaid` (مبلغ کل پرداختی)
- `sellerReceives` (دریافتی فروشنده)
- `platformProfit` (سود سامانه)
